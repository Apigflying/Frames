1.vue实例会代理它内部的data数据 如果data引用的是外部数据，数据类型是引用类型，那么在外部数据改变的时候，会影响vue的数据，从而改变视图，反之，当操作vue的data数据，也会影响外部数据的改变 var data = { a: 1 } var vm = new Vue({ data: data }) vm.a === data.a // -> true // 设置属性也会影响到原始数据 vm.a = 2 data.a // -> 2 // ... 反之亦然 data.a = 3 vm.a // -> 3 总结：vue实例内的data和外部数据会互相影响 2.除了data数据，vue实例暴露了一些游泳的实力属性与方法：这个属性与方法都有前缀$,用来与代理的data属性区分 var data = { a: 1 } var vm = new Vue({ el: '#example', data: data }) vm.$data === data // -> true vm.$el === document.getElementById('example') // -> true // $watch 是一个实例方法 vm.$watch('a', function (newVal, oldVal) { // 这个回调将在 `vm.a` 改变后调用 }) 也就是说，除了 new Vue({ el:"#app", watch:{ }, data:data })这种方式，还可以通过外部的写法 即：var vm=new Vue(); vm.$el=document.getElementById("app")；的方法在外部添加 有一点需要注意：箭头函数，当在new Vue()内使用this的时候，那么均指向实例，如果是写在外部的，那么箭头函数绑定父上下文寻找this就会被改变 3.
